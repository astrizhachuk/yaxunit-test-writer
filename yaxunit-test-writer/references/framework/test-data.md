# Тестовые данные в YAxUnit

Доступ: `ЮТест.Данные()` — возвращает модуль `ЮТТестовыеДанные`.

---

## КонструкторОбъекта — создание объектов ИБ

```bsl
ЮТест.Данные().КонструкторОбъекта("Вид.Имя")
```

Поддерживает справочники, документы, ПВХ, регистры сведений (не подчиненные регистратору). Текучий интерфейс.

### Заполнение реквизитов

| Метод | Описание |
|-------|----------|
| `Установить(ИмяРеквизита, Значение)` | Конкретное значение реквизита |
| `УстановитьРеквизиты(Структура)` | Несколько реквизитов из Структуры |
| `Фикция(ИмяРеквизита, РеквизитыЗаполнения, ОграничениеТипа)` | Случайное валидное значение (примитивные — случайное, ссылочные — новый объект, перечисления — случайное значение) |
| `ФикцияРеквизитов("Рекв1, Рекв2")` | Случайные значения для нескольких реквизитов (строка через запятую или Массив) |
| `ФикцияОбязательныхПолей()` | Заполнить все поля с "Проверка заполнения = Выдавать ошибку" |
| `ФикцияНезаполненных(ИменаРеквизитов)` | Случайные значения для незаполненных реквизитов |
| `ФикцияНезаполненныхИсключая(ИменаРеквизитов)` | Все незаполненные, кроме указанных |

### Табличные части

| Метод | Описание |
|-------|----------|
| `ТабличнаяЧасть(ИмяТЧ)` | Переключиться на табличную часть |
| `ДобавитьСтроку(ЗначенияРеквизитов)` | Добавить строку (параметр — Структура, опционально) |
| `Объект()` | Вернуться к шапке после работы с ТЧ |
| `ДанныеСтроки()` | Данные текущей строки ТЧ |

### Запись

| Метод | Описание |
|-------|----------|
| `Записать(ВернутьОбъект, ОбменДаннымиЗагрузка)` | Записать. `Истина` — вернуть объект, иначе — ссылку. Второй параметр: `Истина` — запись с `ОбменДанными.Загрузка = Истина` |
| `Провести(ВернутьОбъект)` | Записать и провести документ |
| `ДобавитьЗапись(ОбменДаннымиЗагрузка)` | Записать и вернуть конструктор (для цепочки записей) |
| `НовыйОбъект()` | Создать объект без записи в БД |
| `ДанныеОбъекта()` | Данные объекта в виде Структуры |

### Дополнительно

| Метод | Описание |
|-------|----------|
| `УстановитьСсылкуНового(Значение)` | Задать UUID ссылки (УникальныйИдентификатор или Строка) |
| `УстановитьДополнительноеСвойство(ИмяСвойства, Значение)` | Произвольное свойство, не являющееся реквизитом |

**Правило:** используй `Фикция("Реквизит")` вместо `Установить("Реквизит", ЮТест.Данные().СоздатьЭлемент(...))`.

## КонструкторДвижений — создание движений документа

```bsl
ЮТест.Данные().КонструкторДвижений(Документ, ИмяРегистра)
```

Создает движения документа. Работает с регистрами накопления, бухгалтерии и расчета (они всегда подчинены регистратору), а также с регистром сведений — но только если тот подчинен регистратору (например `ЦеныНоменклатуры`). `ИмяРегистра` — строка; допустимо как полное имя (`"РегистрНакопления.ТоварыНаСкладах"`), так и короткое (`"ЦеныНоменклатуры"`). `Документ` — регистратор (ссылка или объект). Текучий интерфейс.

Для регистра сведений, **не** подчиненного регистратору, используй `КонструкторОбъекта` (см. выше).

```bsl
// Регистр сведений, подчиненный регистратору (ЦеныНоменклатуры)
Регистратор = ЮТест.Данные().СоздатьДокумент("Документы.УстановкаЦенНоменклатуры");
ЮТест.Данные().КонструкторДвижений(Регистратор, "ЦеныНоменклатуры")
    .ДобавитьСтроку()
        .Установить("Период", Период)
        .Установить("ТипЦен", ТипЦен)
        .Установить("Номенклатура", Номенклатура)
        .Установить("Цена", Цена)
.Записать();
```

| Метод | Описание |
|-------|----------|
| `ДобавитьСтроку(ЗначенияРеквизитов)` | Добавить запись (Структура, опционально) |
| `Установить(ИмяРеквизита, Значение)` | Значение реквизита текущей записи |
| `УстановитьРеквизиты(Структура)` | Несколько реквизитов |
| `Фикция(ИмяРеквизита, РеквизитыЗаполнения, ОграничениеТипа)` | Фиктивное значение |
| `ФикцияРеквизитов("Рекв1, Рекв2")` | Фиктивные значения для нескольких |
| `ФикцияОбязательныхПолей()` | Обязательные реквизиты записи |
| `ФикцияНезаполненных(ИменаРеквизитов)` | Незаполненные реквизиты |
| `ФикцияНезаполненныхИсключая(ИменаИсключений)` | Все незаполненные кроме указанных |
| `Записать(ОбменДаннымиЗагрузка)` | Зафиксировать движения |
| `ДанныеСтроки()` | Данные текущей записи |
| `Данные()` | Данные всего набора записей |

По умолчанию период = дата документа. Можно задать свой: `Установить("Период", Дата)`.

## Ссылка как значение без создания объекта

Если тесту нужна ссылка только как значение для сравнения (а не реальная запись в БД), используй `ПолучитьСсылку()`:

```bsl
Адрес = Справочники.новаАдреса.ПолучитьСсылку();
```

Это дешевле, чем `КонструкторОбъекта(...).Записать()` — объект не создаётся, транзакция не нужна. Подходит когда тестируемый код лишь сравнивает переданное значение с реквизитом (например, `ГДЕ Т.АдресОбъекта = &АдресОбъекта`). Или когда кроме ссылки из объекта ничего не нужно, или когда нет обращения к объекту.

## Генераторы случайных данных

### Числа

```bsl
ЮТест.Данные().СлучайноеЧисло(Мин, Макс)
ЮТест.Данные().СлучайноеПоложительноеЧисло(Макс)
ЮТест.Данные().СлучайноеОтрицательноеЧисло(Мин)
```

### Строки

```bsl
ЮТест.Данные().СлучайнаяСтрока(Длина)
ЮТест.Данные().СлучайныйИдентификатор(Длина, Префикс)
ЮТест.Данные().СлучайныйIPАдрес()
ЮТест.Данные().СлучайныйНомерТелефона()
ЮТест.Данные().УникальнаяСтрока()              // на базе UUID
```

### Даты

```bsl
ЮТест.Данные().СлучайнаяДата()
ЮТест.Данные().СлучайноеВремя()
ЮТест.Данные().СлучайнаяДатаВБудущем(Количество, Единица)   // "часа", "дня", "месяца"
ЮТест.Данные().СлучайнаяДатаВПрошлом(Количество, Единица)
ЮТест.Данные().СлучайнаяДатаПосле(Дата, Количество, Единица)
ЮТест.Данные().СлучайнаяДатаДо(Дата, Количество, Единица)
```

**Фиксированную дату задавай литералом, а не конструктором `Дата(...)`** — литерал читается лучше и не плодит предупреждения о «магических числах»:

```bsl
// ❌ Плохо — конструктор с набором чисел
ДатаИсходная = Дата(2021, 05, 10, 09, 50, 00);

// ✅ Хорошо — литерал 'ГГГГ-ММ-ДД чч:мм:сс'
ДатаИсходная = '2021-05-10 09:50:00';
```

**Исключение — пустая дата:** `Дата(1, 1, 1)` оставляй конструктором. Это устоявшаяся практика, читается проще литерала `'0001-01-01'`.

### Булево и из списка


## Поиск данных

| Метод | Описание |
|-------|----------|
| `СсылкаПоРеквизиту(МетаданныеОбъекта, ИмяРеквизита, ЗначениеРеквизита)` | Ссылка по значению реквизита |
| `СлучайнаяСсылка(МетаданныеОбъекта, Отборы)` | Случайная ссылка. Отборы: Предикат, Структура, Соответствие, Массив |
| `Ссылка(МетаданныеОбъекта, Значение, Отборы)` | Поиск по стандартным реквизитам (Код, Наименование) |

Формат `МетаданныеОбъекта`: строка `"Справочник.Контрагенты"`, менеджер `Справочники.Контрагенты`, объект метаданных, Тип.

## Удаление тестовых данных

### Транзакции (откат всех изменений)

```bsl
ЮТТесты.ДобавитьТест("ИмяТеста").ВТранзакции()       // конкретный тест
ЮТТесты.ДобавитьТестовыйНабор("Набор").ВТранзакции()  // набор
ЮТТесты.ВТранзакции()                                  // все тесты модуля
```

Оборачивается: тест + `ПередКаждымТестом`/`ПослеКаждогоТеста`. Данные из `ПередВсемиТестами` НЕ откатываются. Для клиентских тестов игнорируется.

### Механизм удаления

```bsl
ЮТТесты.ДобавитьТест("ИмяТеста").УдалениеТестовыхДанных()
ЮТТесты.ВТранзакции().УдалениеТестовыхДанных()   // совместно
```

Запоминает все созданные через API объекты. Удаляет данные с клиента. НЕ работает для данных, созданных не через API или внутри тестируемого кода.

## Хуки жизненного цикла

```bsl

Процедура ПередТестовымНабором() Экспорт    // перед каждым набором
```

## ЮТест.Контекст() — передача данных между хуком и тестом

```bsl
// В хуке
ЮТест.Контекст().УстановитьЗначение("Ключ", Значение);

// В тесте
Значение = ЮТест.Контекст().Значение("Ключ");
```
