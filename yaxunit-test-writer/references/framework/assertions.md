# Утверждения YAxUnit

## Точка входа

```bsl
ЮТест.ОжидаетЧто(ПроверяемоеЗначение, "Пояснение к ошибке")
```

Возвращает модуль `ЮТУтверждения` с текучим (fluent) интерфейсом. Второй аргумент необязателен.

---

## Навигация по объекту

| Метод | Описание |
|-------|----------|
| `Свойство(ИмяСвойства)` | Перейти к свойству. Поддерживает вложенные пути |
| `Элемент(Индекс)` | Алиас для `Свойство` — навигация к элементу коллекции |
| `Объект()` | Сбросить позицию на корневой объект |
| `НетСвойства(ИмяСвойства)` | Утвердить отсутствие свойства и вернуться к корню |

**Важно:** `Свойство` всегда разрешается от корневого объекта, переданного в `ОжидаетЧто`, а НЕ от текущей позиции.

### Поддерживаемые выражения путей

```bsl
.Свойство("Реквизит")                    // прямое свойство
.Свойство("Реквизит.ВложенныйРеквизит")  // вложенное свойство
.Свойство("[0].Товар")                    // элемент коллекции
.Свойство("Товары[0].Товар")              // свойство элемента коллекции
.Свойство("Товары[-1].Сумма")             // последний элемент (отрицательный индекс)
```

## Равенство и сравнение

```bsl
.Равно(Значение)
.НеРавно(Значение)
.Больше(Значение)
.БольшеИлиРавно(Значение)
.Меньше(Значение)
.МеньшеИлиРавно(Значение)
.ВСписке(Коллекция)
```

**Особенности сравнения:**
- Типы сравниваются строго: `1` НЕ равно `Истина` (в отличие от поведения платформы)
- Глубокое сравнение "по значению" для Структур, Соответствий, Массивов, ТаблицЗначений
- `ХранилищеЗначения` распаковывается перед сравнением

```bsl
.МеждуВключаяГраницы(Мин, Макс)          // [Мин, Макс]
```

## Заполненность и существование


## Проверка исключений

Настройка цепочки: `ЮТест.ОжидаетЧто(Объект).Метод("ИмяМетода").Параметр(Знач1).Параметр(Знач2)`

```bsl
.Метод(ИмяМетода, Параметры)                 // задать имя метода; Параметры — массив (опционально)
.Параметр(Значение)                           // добавить параметр к вызову
.ВыбрасываетИсключение(ОжидаемыйТекст)       // метод должен выбросить исключение, текст которого СОДЕРЖИТ подстроку
.НеВыбрасываетИсключение()                    // метод не должен выбрасывать исключение
```

`Объект` — общий модуль, менеджер (`РегистрыСведений.X`, `Документы.X`), объект или экземпляр; тип объекта на работу механизма не влияет.

**`.ВыбрасываетИсключение(ОжидаемыйТекст)` — аргумент обязателен** (у параметра нет значения по умолчанию):
- вызов без аргумента падает с `Недостаточно фактических параметров`, и ошибка в трейсе привязана к строке `ЮТест.ОжидаетЧто(...)` — выглядит как проблема тестируемого метода, но на деле не хватает аргумента самому `ВыбрасываетИсключение`;
- совпадение **подстрочное** (`СтрНайти(ТекстИсключения, ОжидаемыйТекст) > 0`), не точное и не по префиксу — передавай устойчивый фрагмент реального текста в `НСтр`, а не всю формулировку;
- пустую строку `""` передать нельзя — `СтрНайти(..., "")` = 0, проверка всегда провалится.

У `.НеВыбрасываетИсключение()` аргумент необязателен (есть значение по умолчанию) — вызывается без параметров.

**Важно:** Никогда не вызывай метод напрямую — `Метод(Документ.Записать(...))` — это ошибка. Движок тестирования сам вызывает метод.

## ОжидаетЧтоТаблицаБазы — проверка данных в БД

```bsl
ЮТест.ОжидаетЧтоТаблицаБазы(ИмяТаблицы, ОписаниеПроверки)
```

Формат имени таблицы: `"Справочник.Товары"`, `"РегистрСведений.КурсыВалют"`, `"Документ.ЗаказПокупателя"`.

| Метод | Описание |
|-------|----------|
| `СодержитЗаписи(Предикат)` | Таблица содержит записи по условию |
| `НеСодержитЗаписи(Предикат)` | Таблица не содержит записей по условию |
| `СодержитЗаписиСНаименованием(Наименование)` | Есть записи с именем |
| `СодержитЗаписиСКодом(Код)` | Есть записи с кодом |
| `СодержитЗаписиСНомером(Номер)` | Есть записи с номером |
| `НеСодержитЗаписиСНаименованием(Наименование)` | Нет записей с именем |
| `НеСодержитЗаписиСКодом(Код)` | Нет записей с кодом |
| `НеСодержитЗаписиСНомером(Номер)` | Нет записей с номером |

**Рекомендация:** НЕ использовать `ОжидаетЧтоТаблицаБазы` для проверки значений полей. Вместо этого:
1. `ОжидаетЧтоТаблицаБазы` — подтвердить что запись существует
2. `ЮТЗапросы.Запись(ИмяТаблицы, Условие)` — получить запись
3. `ЮТест.ОжидаетЧто(Запись)` — проверить отдельные поля

## Предпочитай методы утверждения вычислениям в BSL

Если у текучего интерфейса есть метод-проверка для нужной характеристики — используй его, а не вычисляй эту характеристику в BSL ради проверки примитива. Такой метод читается как одна цепочка и точнее выражает намерение:

```bsl
// ❌ Хуже — вычисление + примитив, три отдельных ОжидаетЧто
ЮТест.ОжидаетЧто(ТипЗнч(Результат)).Равно(Тип("Структура"));
ЮТест.ОжидаетЧто(Результат.Количество()).Равно(3);
ЮТест.ОжидаетЧто(Результат.Свойство("Штрихкод")).ЭтоИстина();

// ✅ Лучше — одна цепочка утверждений
ЮТест.ОжидаетЧто(Результат)
    .ИмеетТип("Структура")
    .ИмеетДлину(3)
    .ИмеетСвойство("Штрихкод");
```

Частые соответствия: `ТипЗнч()` + `Равно` → `ИмеетТип`; `.Количество()` + `Равно` → `ИмеетДлину`; `.Свойство()` + `ЭтоИстина` → `ИмеетСвойство` / `НеИмеетСвойства`.

**Это предпочтение, а не запрет.** Готового метода-проверки на нужную характеристику может не быть — тогда вычисление в BSL с проверкой примитива уместно и нормально (например, значение надо сперва получить своим методом, или характеристика, для которой готовой проверки нет). Правило срабатывает только когда такой метод существует и покрывает проверку; не изобретай обходные пути, чтобы избежать вычисления там, где без него не обойтись.

## Реальные примеры из проекта

### Проверка типа и значения

```bsl
ЮТест.ОжидаетЧто(Результат)
    .ИмеетТип("Число")
    .Равно(83.33);
```

### Проверка структуры с вложенными свойствами

```bsl
ЮТест.ОжидаетЧто(Результат)
    .ИмеетТип("Структура")
    .Свойство("ДатаДокумента")
        .ИмеетТип("Дата")
        .НеЗаполнено()
    .Свойство("ИнформационнаяБаза")
        .Равно(ИмяИнформационнойБазы)
    .Свойство("Товары")
        .ИмеетТип("Соответствие")
        .НеЗаполнено();
```

### Проверка массива с предикатами

```bsl
ЮТест.ОжидаетЧто(Результат)
    .ИмеетТип("ФиксированныйМассив")
    .ИмеетДлину(2)
    .Содержит(
        ЮТест.Предикат()
            .Реквизит("Код").Равно(Товары[0].sku)
            .Реквизит("ДлиныОтрезков").Заполнено())
    .Содержит(
        ЮТест.Предикат()
            .Реквизит("Код").Равно(Товары[1].sku)
            .Реквизит("ДлиныОтрезков").Заполнено());
```

### Проверка таблицы значений с индексами строк

```bsl
ЮТест.ОжидаетЧто(Результат)
    .ИмеетТип("ТаблицаЗначений")
    .Свойство("[0].Краска").Равно(Краска1.Ссылка)
    .Свойство("[0].ВидКолеровки").Равно(Перечисления.ВидыКолеровкиНоменклатуры.КолеровкаВозможна)
    .Свойство("[0].Объем").Равно(5)
    .Свойство("[0].КолеровкаПоУмолчанию").ЭтоЛожь()
    .Свойство("[1].Краска").Равно(Краска2.Ссылка);
```

### Проверка исключений

```bsl
ЮТест.ОжидаетЧто(ЦелевыеТоварныеЗапасы)
    .Метод("Сформировать", ЮТКоллекции.ЗначениеВМассиве(ОписаниеНастройки))
    .ВыбрасываетИсключение(НСтр("ru = 'Не найдены параметры'"));
```

### Проверка свойств структуры одним вызовом

```bsl
ЮТест.ОжидаетЧто(Результат).ИмеетСвойстваРавные(Ожидание);
```

### Проверка таблицы базы

```bsl
ЮТест.ОжидаетЧтоТаблицаБазы("РегистрСведений.ЦелевыеОстаткиПодразделений")
    .НеСодержитЗаписи();
```
