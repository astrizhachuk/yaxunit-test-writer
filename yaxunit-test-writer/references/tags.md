# Теги в YAxUnit

Теги — механизм группировки и фильтрации тестов при запуске. Тег можно повесить на **модуль**, **тестовый набор** или **отдельный тест** — выбирай уровень, который точнее описывает область.

Два способа задать теги:
- **`.Тег("Один")`** — цепочкой, по одному
- **Параметр `ТегиСтрокой`** в `ДобавитьТест` / `ДобавитьТестовыйНабор` — через запятую: `"Интеграционный, Критичный"`

## Тег проекта/инициативы

Если тесты пишутся в рамках проекта/инициативы, то **обязательно** ставь тег — он позволяет запустить только тесты по этому тегу. Если определить такой из контекста невозможно — проектный тег не нужен.

Проектный тег имеет наименьший приоритет при принятии решений о структуре — **не должен являться критерием для переработки всей структуры тестов**.

## Уровни тегирования

Ставь тег на уровне, где он точно описывает **все** элементы ниже. Если поставить тег выше, чем нужно, он распространится на тесты, для которых неактуален — фильтрация по нему притянет лишнее. **Не дублируй тег на каждый тест, если он уже стоит на наборе или модуле.**

**На модуле** — когда тег актуален для всех тестов модуля:

```bsl
// Все тесты модуля интеграционные и относятся к одному проекту
ЮТТесты
    .Тег("srs-148, Интеграционный")
    .ВТранзакции()
        .ДобавитьТест("ЗаписьСохраняетДанные")
        .ДобавитьТест("ЗаписьПерезаписываетСуществующую")
    ;
```

**На тестовом наборе** — когда тег актуален для всех тестов набора, но не для всего модуля:

```bsl
// Набор "Запись" — все интеграционные; набор "Валидация" — юниты; теги разные → на наборе
ЮТТесты
    .ДобавитьТестовыйНабор("Запись", "srs-571, Интеграционный")
        .ВТранзакции()
            .ДобавитьТест("ЗаписьСохраняетДанные")        // тег унаследован от набора
            .ДобавитьТест("ЗаписьОбновляетСуществующую")  // тег унаследован от набора
    .ДобавитьТестовыйНабор("Валидация", "srs-148")
        .ДобавитьТест("ПроверкаЗаполненияВозвращаетОшибку")
    ;
```

**На отдельном тесте** — когда тег актуален только для конкретного теста:

```bsl
// Внутри набора тесты смешанные: один юнит, один интеграционный → тег только на нужный тест
ЮТТесты
    .ДобавитьТестовыйНабор("Поиск и запись")
        .ДобавитьТест("ПоискВозвращаетПустойРезультат", , "srs-100")       // юнит
        .ДобавитьТест("ЗаписьСохраняетДанные", , "srs-100, Интеграционный")
            .ВТранзакции()
    ;
```

## Типичные теги

| Тег              | Назначение                                            |
| ---------------- | ----------------------------------------------------- |
| `srs-NNN`        | Проект/задача из трекера                              |
| `Интеграционный` | Тест работает с реальной БД, файловые операции и т.д. |
| `Компонентный`   | Компонентный тест (несколько модулей вместе)          |
