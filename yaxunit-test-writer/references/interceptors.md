# Создание мок-перехватчиков

Мокито работает через механизм расширений 1С (`&Вместо`). Чтобы замокировать метод, в тестовом расширении должен существовать файл-перехватчик с соответствующей функцией `&Вместо`.

## Шаг 1 — проверь, существует ли уже перехватчик

Сначала проверь, есть ли уже файл перехватчика для нужного объекта. Путь зависит от типа:

| Тип объекта                        | Путь к файлу перехватчика              | Имя файла           |
| ---------------------------------- | -------------------------------------- | ------------------- |
| Общий модуль                       | `test/CommonModules/ИмяМодуля/Ext/`    | `Module.bsl`        |
| Модуль менеджера документа         | `test/Documents/ИмяДок/Ext/`           | `ManagerModule.bsl` |
| Модуль объекта документа           | `test/Documents/ИмяДок/Ext/`           | `ObjectModule.bsl`  |
| Модуль менеджера справочника       | `test/Catalogs/ИмяСпр/Ext/`            | `ManagerModule.bsl` |
| Модуль объекта справочника         | `test/Catalogs/ИмяСпр/Ext/`            | `ObjectModule.bsl`  |
| Модуль менеджера регистра сведений | `test/InformationRegisters/ИмяРС/Ext/` | `ManagerModule.bsl`  |
| Модуль набора записей регистра     | `test/InformationRegisters/ИмяРС/Ext/` | `RecordSetModule.bsl` |

- **Файл существует** → просто добавь новую функцию `&Вместо` в конец файла. XML и `Configuration.xml` не трогай.
- **Файла нет** → создай перехватчик: BSL + XML + регистрация в `Configuration.xml` (шаги 2–4).

## Шаг 2 — BSL-файл перехватчика

**Шаблон для функции:**

```bsl
&Вместо("ИмяМетода")
Функция Мок_ИмяМетода(Парам1, Парам2) Экспорт

	ПараметрыМетода = Мокито.МассивПараметров(Парам1, Парам2);

	ПрерватьВыполнение = Ложь;
	Результат = МокитоПерехват.АнализВызова(ОбъектМодуля, "ИмяМетода", ПараметрыМетода, ПрерватьВыполнение);

	Если НЕ ПрерватьВыполнение Тогда
		Возврат ПродолжитьВызов(Парам1, Парам2);
	Иначе
		Возврат Результат;
	КонецЕсли;

КонецФункции
```

**Шаблон для процедуры:**

```bsl
&Вместо("ИмяПроцедуры")
Процедура Мок_ИмяПроцедуры(Парам1) Экспорт

	ПараметрыМетода = Мокито.МассивПараметров(Парам1);

	ПрерватьВыполнение = Ложь;
	МокитоПерехват.АнализВызова(ОбъектМодуля, "ИмяПроцедуры", ПараметрыМетода, ПрерватьВыполнение);

	Если НЕ ПрерватьВыполнение Тогда
		ПродолжитьВызов(Парам1);
	КонецЕсли;

КонецПроцедуры
```

**Правила:**
- `&Вместо("ИмяМетода")` — имя метода в оригинальном модуле, строка
- Имя функции: `Мок_[ИмяМетода]`
- Параметры — точно как в оригинале (включая `Знач` и значения по умолчанию)
- `Экспорт` — только если оригинальный метод экспортный; обработчики событий объекта (`ПередЗаписью`, `ПриЗаписи` и т.д.) не экспортные — `Экспорт` не нужен
- `ОбъектМодуля` в `АнализВызова` — переменная-ссылка на объект, не строка:
  - Общий модуль: `ИмяМодуля`
  - Менеджер документа: `Документы.ИмяДок`
  - Менеджер справочника: `Справочники.ИмяСпр`
  - Менеджер регистра: `РегистрыСведений.ИмяРС`
  - Модуль объекта документа или справочника: `ЭтотОбъект` (экземпляр, не менеджер)
  - Набор записей регистра сведений: `ЭтотОбъект` (экземпляр набора записей)
- Для **модуля объекта** мок в тесте настраивается на конкретный экземпляр, а не на менеджер:

  ```bsl
  // Менеджер:
  Мокито.Обучение(Документы.ЗаказПокупателя).Когда("МойМетод").Вернуть(42).Прогон();
  // Объект:
  Объект = Документы.ЗаказПокупателя.СоздатьДокумент();
  Мокито.Обучение(Объект).Когда("МойМетод").Вернуть(42).Прогон();
  ```

- `ПродолжитьВызов(...)` — передаёт те же параметры что получили, не добавлять лишних

**Пример с несколькими методами в одном файле:**

```bsl
&Вместо("ЭтоКолеровка")
Функция Мок_ЭтоКолеровка(Знач Услуга) Экспорт

	ПараметрыМетода = Мокито.МассивПараметров(Услуга);

	ПрерватьВыполнение = Ложь;
	Результат = МокитоПерехват.АнализВызова(КолеровкаПовтИсп, "ЭтоКолеровка", ПараметрыМетода, ПрерватьВыполнение);

	Если НЕ ПрерватьВыполнение Тогда
		Возврат ПродолжитьВызов(Услуга);
	Иначе
		Возврат Результат;
	КонецЕсли;

КонецФункции

&Вместо("ПолучитьУслугиКолеровки")
Функция Мок_ПолучитьУслугиКолеровки() Экспорт

	ПараметрыМетода = Мокито.МассивПараметров();

	ПрерватьВыполнение = Ложь;
	Результат = МокитоПерехват.АнализВызова(КолеровкаПовтИсп, "ПолучитьУслугиКолеровки", ПараметрыМетода, ПрерватьВыполнение);

	Если НЕ ПрерватьВыполнение Тогда
		Возврат ПродолжитьВызов();
	Иначе
		Возврат Результат;
	КонецЕсли;

КонецФункции
```

## Шаг 3 — XML-метаданные (только при создании нового файла)

Перехватчик — это **adopted**-расширение существующего объекта основной конфигурации.

### Общий принцип построения XML

Структура XML перехватчика строится на основе `src/[Тип]/[ИмяОбъекта].xml`:

1. **Корневой тег** (`<CommonModule>`, `<Document>`, `<Catalog>`, `<InformationRegister>`) — новый UUID.
2. **`<InternalInfo>`** — скопировать блок `<xr:GeneratedType>` из src, **заменив все UUID (TypeId и ValueId) на новые**; структура (имена, категории, порядок) остаётся точно такой же. Добавить в конец блока один или несколько блоков `<xr:PropertyState>` — по одному на каждый расширяемый модуль:
   - Для общего модуля: `<xr:Property>Module</xr:Property>`
   - Для модуля менеджера: `<xr:Property>ManagerModule</xr:Property>`
   - Для модуля объекта: `<xr:Property>ObjectModule</xr:Property>`
   - Для набора записей регистра сведений: `<xr:Property>RecordSetModule</xr:Property>`

   Если расширение перекрывает несколько модулей, добавь соответствующие блоки `<xr:PropertyState>` подряд.
3. **`<Properties>`** — минимальный набор: `<Name>`, `<Comment/>`, `<ObjectBelonging>Adopted</ObjectBelonging>`, `<ExtendedConfigurationObject>{uuid корневого тега из src}</ExtendedConfigurationObject>`. Для регистров сведений дополнительно скопировать `<InformationRegisterPeriodicity>` и `<WriteMode>` из src.
4. Общие модули не имеют `<xr:GeneratedType>` ни в src, ни в test — `<InternalInfo>` содержит только `<xr:PropertyState>`.

Для генерации UUID на Windows:
```bash
# Один UUID
powershell -Command "[guid]::NewGuid().ToString()"
# Сразу несколько (например 11 для документа/справочника, 15 для регистра)
powershell -Command "1..15 | % { [guid]::NewGuid().ToString() }"
```

### Шаблон для общего модуля

Нужен **1 UUID** (корневой тег). Свойства `ClientManagedApplication`, `ExternalConnection`, `ClientOrdinaryApplication`, `ServerCall` скопировать из `src/CommonModules/[ИмяМодуля].xml`.

Прочитай шаблон из `templates/{версия-платформы}/common-module-adopted.xml` и подставь значения: `{новый-uuid}` → новый UUID, `[ИмяМодуля]` → имя модуля, `{uuid-из-src-xml}` → UUID корневого тега из `src`, `???` → значения из src.

## Шаг 4 — регистрация в `Configuration.xml` (только при создании нового файла)

Добавь в `<ChildObjects>` в **алфавитном порядке** тег, соответствующий типу объекта:

```xml
<CommonModule>ИмяМодуля</CommonModule>
<!-- или -->
<Document>ИмяДокумента</Document>
<!-- или -->
<Catalog>ИмяСправочника</Catalog>
<!-- или -->
<InformationRegister>ИмяРегистра</InformationRegister>
```

Adopted-объекты регистрируются так же, как новые — просто по имени, без специальных атрибутов.
