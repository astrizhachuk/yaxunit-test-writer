# Контексты выполнения и директивы компиляции

Применяется только для тестовых модулей с `ClientManagedApplication=true` — тех, что компилируются и на клиенте, и на сервере.

Главное требование: **модуль должен компилироваться в каждом из своих контекстов**. Директивы компиляции ставятся внутри тела процедур — не вокруг самих определений процедур.

Внутри клиент-серверного модуля могут быть тесты трёх видов:

- `ДобавитьТест` — YAxUnit выполняет тест во всех контекстах модуля. Код процедуры компилируется и на клиенте, и на сервере; серверно-зависимые блоки оборачиваются в `#Если Сервер Тогда ... #Иначе ... #КонецЕсли` с клиентской альтернативой.
- `ДобавитьСерверныйТест` — YAxUnit выполняет тест только на сервере. Процедура не оборачивается целиком — только конкретные серверно-зависимые строки внутри тела заключаются в `#Если Сервер Тогда ... #КонецЕсли` (без `#Иначе`), так как модуль по-прежнему компилируется на клиенте.
- `ДобавитьКлиентскийТест` — YAxUnit выполняет тест только на клиенте. Прямой доступ к объектам метаданных как значениям (`Справочники.*`, `РегистрыСведений.*`) недоступен — для ссылок используй `ПредопределенноеЗначение()`, для серверных операций — `ТестовыеДанныеВызовСервера`.

> **Примечание о Мокито.** Модуль `Мокито` скомпилирован для обоих контекстов: `Мокито.Обучение()`, `Мокито.Проверить()` и вспомогательные функции доступны и на сервере, и на клиенте. Ограничение — в **аргументах**: объекты `Справочники.*`, `РегистрыСведений.*` и другие менеджеры метаданных существуют только на сервере и не могут быть переданы в Мокито на клиенте напрямую. На клиенте вместо этого используй `ТестовыеДанныеВызовСервера.МокитоОбучениеВернуть()`, который принимает имя объекта строкой и вызывает Мокито на сервере.

## `ДобавитьТест` — обе ветви через `#Если ... #Иначе`

```bsl
Процедура МетодРаботаетВОбоихКонтекстах() Экспорт
    // given
#Если Сервер Тогда
    Товар = Справочники.Номенклатура.Доставка;
    Мокито.Обучение(РегистрыСведений.ЗначенияСвойствОбъектов)
        .Когда("НайтиОбъектыПоСвойствуИЗначению")
            .Вернуть(ЮТКоллекции.ЗначениеВМассиве(Товар))
        .Прогон();
#Иначе
    Товар = ПредопределенноеЗначение("Справочник.Номенклатура.Доставка");
    ТестовыеДанныеВызовСервера.МокитоОбучениеВернуть(
        "РегистрыСведений.ЗначенияСвойствОбъектов",
        "НайтиОбъектыПоСвойствуИЗначению",
        ЮТКоллекции.ЗначениеВМассиве(Товар));
#КонецЕсли

    // when
    Результат = МойМодуль.МойМетод(Товар);

    // then
    ЮТест.ОжидаетЧто(Результат).ЭтоИстина();
КонецПроцедуры
```

## `ДобавитьСерверныйТест` — серверные строки без `#Иначе`, остальной код общий

```bsl
Процедура МетодТребуетСервернуюПроверку() Экспорт
    // given
    Товар = Неопределено;
#Если Сервер Тогда
    Товар = Справочники.Номенклатура.Доставка;
    Мокито.Обучение(Справочники.ПравилаПодбораБарабанов)
        .Когда("ПолучитьДиапазоны")
            .Вернуть(Новый ФиксированныйМассив(Новый Массив()))
        .Прогон();
#КонецЕсли

    // when
    Результат = МойМодуль.МойМетод(Товар);

    // then
    ЮТест.ОжидаетЧто(Результат).ИмеетТип("ФиксированныйМассив");
#Если Сервер Тогда
    Мокито.Проверить(Справочники.ПравилаПодбораБарабанов)
        .КоличествоВызовов("ПолучитьДиапазоны")
        .Равно(1);
#КонецЕсли
КонецПроцедуры
```

## Ключевые различия между контекстами

- **Сервер**: прямой доступ к объектам метаданных (`Справочники.*`, `РегистрыСведений.*`) как значениям — для передачи в методы и в `Мокито.Обучение()`
- **Клиент**: ссылки на предопределённые значения через `ПредопределенноеЗначение("Справочник.Номенклатура.ПустаяСсылка")`, мокирование через `ТестовыеДанныеВызовСервера.МокитоОбучениеВернуть()` (принимает имя объекта строкой)
